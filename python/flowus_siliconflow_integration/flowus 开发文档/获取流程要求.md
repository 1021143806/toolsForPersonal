# 大致流程
先从配置的页面id获取块内容，其中包括一段提示词和一张叫做“日记”的多维表
该多维表中包含了每日记录的信息，其中的页面含有页面链接
将日记获取的信息单独放到一个数据库表中
然后按照其中包含的页面链接进行获取，获取需要获取页面信息和页面块内容，即两个接口，保持在另一个数据库表中，需要区分是问题记录表还是项目记录表，也使用数据库表区分。

信息获取完后再根据配置文件中配置的要求最近日期，从数据库获取指定时间内（如30天内为当前日期前30天内的信息）的相关信息，保存到outputs中的data.log，然后发送给硅基api生成内容。生成的内容在flowus中有一份，本地则保存在outputs/local.md

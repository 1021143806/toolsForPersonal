# 项目目录重构迁移说明

## 迁移概述

本次重构对整个项目的根目录进行了重新组织，将文件按类型和功能分类到不同的目录中，提高了项目的可维护性和组织性。

## 新的目录结构

```
toolsForPersonal/
├── projects/                    # 所有项目目录
│   ├── flowus_integration/      # FlowUs AI内容生成项目
│   ├── agv_system/             # AGV设备管理系统
│   ├── file_upload/            # 文件上传系统
│   └── web/                    # Web相关项目
├── scripts/                     # 脚本目录
│   ├── powershell_scripts/      # PowerShell脚本
│   └── shell_scripts/          # Shell脚本
├── docs/                        # 文档目录
│   └── readme_files/           # README文档
├── .gitignore                   # Git忽略文件
├── nginx.htaccess              # Nginx配置
├── output.txt                  # 输出文件
├── README.en.md                # 英文README
└── README.md                   # 中文README
```

## 目录迁移详情

### 1. projects/ 目录

#### flowus_integration/
- **原位置**: `python/flowus_siliconflow_integration/`
- **新位置**: `projects/flowus_integration/flowus_siliconflow_integration/`
- **说明**: FlowUs AI内容生成器，集成了FlowUs API和SiliconFlow API

#### agv_system/
- **原位置**: `main/`
- **新位置**: `projects/agv_system/main/`
- **说明**: AGV设备管理和统计系统

#### file_upload/
- **原位置**: `文件上传/`
- **新位置**: `projects/file_upload/`
- **说明**: 文件上传功能模块

#### web/
- **原位置**: `web/`
- **新位置**: `projects/web/`
- **说明**: Web相关项目

### 2. scripts/ 目录

#### powershell_scripts/
- **原位置**: `powershell/`
- **新位置**: `scripts/powershell_scripts/`
- **说明**: 所有PowerShell脚本，包括：
  - 数据库连接脚本
  - curl命令脚本
  - 系统管理脚本
  - 窗体应用脚本

#### shell_scripts/
- **原位置**: `sh/`
- **新位置**: `scripts/shell_scripts/`
- **说明**: Shell脚本集合

### 3. docs/ 目录

#### readme_files/
- **原位置**: 根目录下的README文件
- **新位置**: `docs/readme_files/`
- **说明**: 项目文档和说明文件

## 配置文件处理

按照要求，所有来自git clone的项目配置文件都保持在原位置，没有进行合并：

- FlowUs项目的配置文件仍在 `projects/flowus_integration/flowus_siliconflow_integration/` 中
- AGV系统的配置文件仍在相应项目目录中
- 各项目的独立配置保持不变

3. **脚本执行**: PowerShell和Shell脚本的执行路径需要相应调整

## 迁移后的优势

1. **更好的组织性**: 按类型和功能分类，便于查找和维护

2. **清晰的职责分离**: 项目、脚本、文档分别管理

3. **可扩展性**: 新项目可以轻松添加到相应目录

4. **保持独立性**: 各项目配置和依赖关系保持不变

## 使用建议

   - 脚本工具 → `scripts/`
   - 文档资料 → `docs/`

2. 定期整理各目录内容，保持结构清晰

3. 更新相关文档时，请同步更新迁移说明

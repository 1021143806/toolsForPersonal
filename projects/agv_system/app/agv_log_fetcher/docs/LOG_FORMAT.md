# AGV系统日志格式说明

本文档详细说明了AGV系统中各类日志的文件格式、目录结构和内容格式。

## 目录结构

### 1. TPS日志目录
```
/main/app/tps/logs/
├── TPS.log                     # 当前正在使用的TPS日志
├── TPS-YYYY-MM-DD.1.log       # 历史分片日志1
├── TPS-YYYY-MM-DD.2.log       # 历史分片日志2
└── ...                        # 更多分片日志
```

### 2. ICS日志目录
```
/main/app/ics/logs/
├── ICS.log                     # 当前正在使用的ICS日志
├── ICS-YYYY-MM-DD.1.log       # 历史分片日志1
├── ICS-YYYY-MM-DD.2.log       # 历史分片日志2
└── ...                        # 更多分片日志
```

### 3. RTPS算法目录
```
/main/app/
├── rtpsa-4,5,6,8,16,17,18/    # RTPS算法文件夹（包含区域4,5,6,8,16,17,18）
│   ├── TAL_log/               # TAL调度日志
│   │   ├── dispatchTAL.log     # 调度日志文件
│   │   ├── tal_log_202604011111_977.zip   # 压缩的调度日志
│   │   └── ...                # 其他调度日志文件
│   └── logs/                  # RTPS算法日志
│       ├── rtps.log           # 当前RTPS日志
│       ├── rtps.log.20260219125337121.gz  # 历史压缩日志
│       └── ...                # 其他RTPS日志文件
├── rtpsp-4/                   # RTPS规划文件夹（区域4）
│   └── DPL_log/               # DPL日志
│       ├── dispatch4.log      # 区域4的调度日志
│       ├── dis_log_202603042220_821.gz     # 压缩的调度日志
│       └── ...                # 其他调度日志文件
└── ...                        # 其他区域文件夹
```

## 日志文件命名规则

### 1. TPS/ICS分片日志
```
TPS-YYYY-MM-DD.N.log
ICS-YYYY-MM-DD.N.log
```
- `YYYY-MM-DD`: 日志日期
- `N`: 分片编号（从1开始递增）

### 2. RTPS算法日志文件

#### TAL调度日志
```
TAL日志文件：
- dispatchTAL.log                     # 当前调度日志
- tal_log_YYYYMMDDHHMM_毫秒.zip       # 历史压缩日志
- tal_log_YYYYMMDDHHMM_毫秒           # 历史未压缩日志
```

#### RTPS日志
```
RTPS日志文件：
- rtps.log                           # 当前RTPS日志
- rtps.log.YYYYMMDDHHMMSS毫秒.gz     # 历史压缩日志
- rtps.log.YYYYMMDDHHMMSS毫秒        # 历史未压缩日志
```

#### DPL调度日志
```
DPL日志文件：
- dispatch<区域号>.log                # 指定区域的调度日志
- dispatch.log                       # 通用调度日志
- dis_log_YYYYMMDDHHMM_毫秒.gz       # 历史压缩调度日志
- dis_log_YYYYMMDDHHMM_毫秒          # 历史未压缩调度日志
```

## 日志内容格式

### 1. 标准时间戳格式
所有日志文件通常以标准时间戳开始每行日志：

```log
2026-04-01 10:23:45 [INFO] 任务开始执行
2026-04-01 10:23:45 [DEBUG] 接收到任务参数: task_id=1234
2026-04-01 10:23:46 [WARN] 任务执行超时检测
2026-04-01 10:23:47 [ERROR] 任务执行失败: 连接超时
```

**时间戳格式：**
```
YYYY-MM-DD HH:MM:SS
```

### 2. 日志级别
- `[INFO]`: 一般信息
- `[DEBUG]`: 调试信息
- `[WARN]`: 警告信息
- `[ERROR]`: 错误信息
- `[FATAL]`: 致命错误

### 3. 常见日志模式

#### TPS日志示例
```log
2026-04-01 10:15:30 [INFO] TPS启动成功
2026-04-01 10:15:35 [INFO] 接收到新任务: id=5678, type=MOVE
2026-04-01 10:15:40 [DEBUG] 路径规划开始
2026-04-01 10:15:45 [INFO] 路径规划完成，路径点: 5个
```

#### RTPS算法日志示例
```log
2026-04-01 11:09:49 [INFO] RTPS算法初始化完成
2026-04-01 11:09:50 [DEBUG] 接收到调度命令: target=工作站A
2026-04-01 11:09:51 [INFO] 开始避障计算
2026-04-01 11:09:52 [WARN] 检测到障碍物，重新规划路径
```

#### ICS日志示例
```log
2026-04-01 09:45:20 [INFO] ICS服务启动
2026-04-01 09:45:25 [INFO] 设备连接成功: AGV-001
2026-04-01 09:45:30 [ERROR] 设备心跳超时: AGV-002
2026-04-01 09:45:35 [INFO] 设备重连成功: AGV-002
```

## 文件大小和压缩

### 1. 原始日志文件
- **TPS.log/ICS.log**: 通常较大，可能需要分片
- **分片条件**: 文件超过指定大小（如100MB）时自动分片
- **分片编号**: 按时间顺序递增

### 2. 压缩文件
- **压缩格式**: `.gz` (gzip) 或 `.zip`
- **压缩时机**: 定期（如每天）或根据文件大小
- **压缩文件名**: 包含时间戳和毫秒数的唯一标识

## 查找算法说明

### 1. 时间范围匹配
脚本使用以下逻辑确定包含指定时间戳的日志文件：
1. 读取文件的第一行和最后一行
2. 提取时间戳部分
3. 转换为Unix时间戳（秒数）
4. 检查目标时间戳是否在该时间范围内

### 2. 分片文件选择
对于分片日志：
1. 按分片编号排序
2. 依次检查每个分片文件的时间范围
3. 选择第一个包含目标时间戳的文件

### 3. 压缩文件处理
对于压缩文件：
1. 从文件名中提取时间戳信息
2. 比较文件时间与目标时间的接近程度
3. 选择时间最接近的文件

## 异常情况处理

### 1. 文件格式异常
- 如果日志文件没有标准时间戳，脚本会输出警告并尝试其他文件
- 支持多种时间戳格式的解析

### 2. 文件损坏
- 如果文件读取失败，脚本会跳过该文件并尝试其他文件
- 会输出详细的错误信息

### 3. 时间戳解析失败
- 如果时间戳格式无法解析，脚本会尝试其他时间格式
- 如果所有格式都失败，会输出错误信息并退出

## 最佳实践

### 1. 日志分析建议
1. **问题定位**: 先查看ERROR级别的日志
2. **时序分析**: 使用时间戳追踪事件链
3. **关联分析**: 结合不同类型的日志进行分析

### 2. 脚本使用建议
1. **时间点选择**: 选择已知事件发生的时间点
2. **区域号确认**: 确认问题所在的区域号
3. **日志组合**: 同时拉取多种类型日志以便综合分析

### 3. 存储管理
1. **定期清理**: 定期清理旧的拉取日志
2. **命名规范**: 使用统一的时间戳格式命名文件
3. **分类存储**: 按类型和时间分类存储日志文件
